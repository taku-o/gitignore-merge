# 実装計画

- [x] 1. Go プロジェクト基盤のセットアップ
- [x] 1.1 Go モジュールとディレクトリ構造を作成する
  - `go.mod` を初期化し、モジュール名を設定する
  - `cmd/gitignore-merge/`、`internal/parser/`、`internal/merger/`、`test/testdata/`、`docs/` ディレクトリを作成する
  - `go build ./...` がエラーなく成功する状態にする
  - _Requirements: 1.1, 3.1_

- [x] 2. .gitignore ファイルのパース機能を実装する
- [x] 2.1 セクション構造付きパーサーを実装する
  - ファイルを行単位で読み込み、`#` で始まるコメント行をセクションヘッダーとして認識する
  - 連続するコメント行を1つのセクションヘッダーとしてまとめる
  - パターン行をそのセクションに帰属させ、空行も保持する
  - ファイル先頭にヘッダーがない場合、無名セクション（Header が nil）として扱う
  - `ParseFile` に任意のファイルパスを渡すと、セクション構造付きの結果が返る
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_
  - _Boundary: Parser_

- [x] 2.2 パース結果をテキストに変換する Format 関数を実装する
  - `GitignoreFile` を受け取り、元のファイル形式に復元したテキストを返す
  - `ParseFile` → `Format` の往復で元ファイルの内容が保持されることを保証する
  - _Requirements: 1.1_
  - _Boundary: Parser_

- [x] 2.3 パーサーの単体テストを実装する
  - セクション付きファイルのパース結果が期待通りか検証する
  - 無名セクション、空行保持、連続コメント行のヘッダー認識をテストする
  - `ParseFile` → `Format` の往復一貫性テストを作成する
  - 全テストが `go test ./internal/parser/` で成功する
  - _Requirements: 4.1_
  - _Boundary: Parser_

- [x] 3. 複数ファイルのマージ機能を実装する
- [x] 3.1 セクションベースのマージロジックを実装する
  - 先頭ファイルをベースとし、後続ファイルのセクションを順番にマージする
  - セクション名の比較はヘッダーの最初の `#` 行から `#` とスペースを除去した文字列で行う
  - 同名セクションが存在する場合、後続ファイルのパターンをベースのセクションに統合する
  - ベースに存在しないセクションは末尾に追加する
  - _Requirements: 2.1, 2.2, 2.3_
  - _Boundary: Merger_

- [x] 3.2 重複除去と矛盾解決ロジックを実装する
  - 同一セクション内の完全一致する重複パターンを除去する
  - 矛盾パターン（`path` と `!path`）を検出し、ベースファイル側のパターンを優先する
  - 3ファイル以上の逐次マージが正しく動作する
  - _Requirements: 2.4, 2.5_
  - _Boundary: Merger_

- [x] 3.3 マージの単体テストを実装する
  - 同名セクションのマージ、新規セクションの追加を検証する
  - 重複パターンの除去と矛盾パターンの先頭優先を検証する
  - 3ファイル以上の逐次マージの正確性を検証する
  - 全テストが `go test ./internal/merger/` で成功する
  - _Requirements: 4.2, 4.4, 4.5_
  - _Boundary: Merger_

- [x] 4. CLI エントリポイントを実装する
- [x] 4.1 コマンドライン引数の処理とマージ実行を実装する
  - `os.Args[1:]` からファイルパスを取得し、2つ未満の場合は使い方を表示して終了コード 1 で終了する
  - 各ファイルの存在確認を行い、存在しない場合はエラーメッセージを stderr に表示して終了コード 1 で終了する
  - 全ファイルをパースし、マージ処理を呼び出し、結果を標準出力に出力する
  - `go build ./cmd/gitignore-merge/` でバイナリがビルドでき、`gitignore-merge file1 file2` の形式で実行できる
  - _Requirements: 3.1, 3.2, 3.3, 3.4_
  - _Depends: 2.1, 2.2, 3.1, 3.2_
  - _Boundary: Main_

- [ ] 5. 統合テストとテストデータを作成する
- [ ] 5.1 A, B, C のテスト用 .gitignore ファイルと期待結果を作成する
  - 複数セクション、重複パターン、矛盾パターンを含むテストファイル A, B, C を `test/testdata/` に作成する
  - A + B + C のマージ期待結果を `expected_abc.gitignore` として作成する
  - テストファイル自体がマージ動作のドキュメント（具体例）としても機能する内容にする
  - _Requirements: 4.3_
  - _Boundary: Test_

- [ ] 5.2 統合テストを実装する
  - A, B, C の3ファイルをパース・マージし、期待結果ファイルと一致するか検証する
  - `go test ./test/` で統合テストが成功する
  - _Requirements: 4.3_
  - _Depends: 5.1_
  - _Boundary: Test_
